# Domain Glossary — nano-coder

> Ubiquitous language for this project. Code, docs, configs, and agents MUST use these terms exactly.

---

## NanoCoder

**Definition:** Research and tooling project that fine-tunes Small Language Models (SLMs) to generate JS, HTML, and FreeMarker code using synthetic training data produced by a larger teacher LLM.
**Not the same as:** A general-purpose coding assistant product; a hosted API service (phase 1 is local pipeline).
**Code name:** `NanoCoder`

---

## TargetLanguage

**Definition:** One of the three output languages the student model must learn.
**Enum values:** `JavaScript`, `HTML`, `FreeMarker`
**Code name:** `TargetLanguage`

---

## TrainingMethod

**Definition:** Legacy composite tag for a `TrainingRun`. Prefer explicit `CompressionMethod` + `DataSchedule` (ADR-005).
**Enum values (CompressionMethod):** `FewShot`, `FullFineTune`, `LoRA`, `QLoRA`, `DoRA`, `GaLore`, `GradientVarianceAdaptiveLR`, `DataReupload`, `FrozenBackboneHybridHead`, `MPO`, `QTHA`, `TTLoRA`, `TRG`, `iPEPS`, `LowRankSketching`, `DecoherenceRegularization`
**Enum values (DataSchedule):** `MixedLanguages`, `IncrementalByLanguage`, `SingleLanguage`
**Not the same as:** `TeacherModel` (inference-only for data generation).
**Code name:** `TrainingMethod` (composite); see also `CompressionMethod`, `DataSchedule`

---

## CompressionMethod

**Definition:** How trainable parameters or gradients are constrained during fine-tuning.
**Enum values:** See [TRAINING-METHODS.md](TRAINING-METHODS.md) tiers T0–T3.
**Code name:** `CompressionMethod`

---

## DataSchedule

**Definition:** How examples from multiple `TargetLanguage` values are ordered during training.
**Enum values:** `MixedLanguages`, `IncrementalByLanguage`, `SingleLanguage`
**Code name:** `DataSchedule`

---

## BondDimension

**Definition:** Tensor-network compression knob χ (virtual bond size). Central hyperparameter for `MPO`, `QTHA`, `TTLoRA`, `TRG`, `iPEPS`. Analogous role to `loraRank` but not interchangeable in reports.
**Enum values (sweep):** `4`, `8`, `16`, `32`
**Code name:** `BondDimension`

---

## EvidenceLevel

**Definition:** Honest classification of how well-supported a `CompressionMethod` is in literature for our context.
**Enum values:** `Established`, `QuantumInspired`, `NovelApplication`, `Speculative`
**Code name:** `EvidenceLevel`

---

## DecoherenceScore

**Definition:** Speculative (L4) regularization metric — activation entropy vs a low-entropy reference during fine-tune. Used only with `DecoherenceRegularization`.
**Code name:** `DecoherenceScore`

---

## GradientVarianceAdaptiveLR

**Definition:** Adaptive optimizer scaling learning rate from gradient variance (GV-ALR). Proven in quantun-ia exp_054/075 for training efficiency at parity accuracy.
**Not the same as:** `GaLore` (subspace projection) or fixed Adam.
**Code name:** `GradientVarianceAdaptiveLR`

---

## DataReupload

**Definition:** Multi-pass training schedule re-injecting representations between layers/blocks — transferred from quantun-ia re-upload QNN (exp_008, exp_001 significant wins).
**Not the same as:** `IncrementalByLanguage` (language ordering, not layer re-injection).
**Code name:** `DataReupload`

---

## FrozenBackboneHybridHead

**Definition:** Freeze full student backbone; train adapters on head + last blocks only. Pattern from quantun-ia hybrid QNN head (exp_037/051/062/071 — 4/4 anchors accepted).
**Not the same as:** Full `LoRA` on all layers.
**Code name:** `FrozenBackboneHybridHead`

---

## TeacherModel

**Definition:** Large LLM (API or local) that generates synthetic instruction–code pairs and optionally acts as LLM-as-judge.
**Not the same as:** `StudentModel` (the model being trained/evaluated).
**Code name:** `TeacherModel`

---

## StudentModel

**Definition:** Small Language Model (sub-1B to ~3B parameters) that receives fine-tuning on synthetic data and is evaluated in the benchmark.
**Enum values (candidates):** `Qwen2.5-Coder-0.5B`, `Qwen2.5-Coder-1.5B`, `Gemma3-270M`, `Gemma3-1B`, `SmolLM3` (final choice via ADR).
**Code name:** `StudentModel`

---

## GoldSeedExample

**Definition:** Manually authored instruction–code pair of reference quality used to bootstrap synthetic expansion.
**Not the same as:** `SyntheticExample` (machine-generated from seeds).
**Code name:** `GoldSeedExample`

---

## SyntheticExample

**Definition:** A single instruction–code pair generated by the teacher model, subject to quality filtering before inclusion in a dataset.
**Code name:** `SyntheticExample`

---

## SyntheticDataset

**Definition:** Versioned collection of validated `SyntheticExample` records for one or more `TargetLanguage` values, ready for training.
**Not the same as:** Raw generation output before filtering.
**Code name:** `SyntheticDataset`

---

## QualityGate

**Definition:** Automated or LLM-based check that accepts or rejects a `SyntheticExample` before it enters a `SyntheticDataset`.
**Enum values (gate types):** `SyntaxValidation`, `LintPass`, `ExecutionCheck`, `LLMJudge`, `ManualReviewSample`
**Code name:** `QualityGate`

---

## DatasetGenerationRun

**Definition:** A bounded execution that expands seeds into candidate examples, applies quality gates, and produces or updates a `SyntheticDataset`.
**Code name:** `DatasetGenerationRun`

---

## TrainingRun

**Definition:** A bounded execution that fine-tunes a `StudentModel` on a `SyntheticDataset` using a specific `TrainingMethod`.
**Code name:** `TrainingRun`

---

## BenchmarkRun

**Definition:** A bounded evaluation that scores one trained (or baseline) `StudentModel` against a held-out test set across metrics defined in `EVALUATION-METHOD.md`.
**Code name:** `BenchmarkRun`

---

## HeldOutTestSet

**Definition:** Fixed set of tasks never used during training or synthetic generation; used only in `BenchmarkRun`.
**Not the same as:** Training split of `SyntheticDataset`.
**Code name:** `HeldOutTestSet`

---

## PassAtK

**Definition:** Metric: probability that at least one of *k* generated samples passes correctness checks for a task.
**Code name:** `PassAtK`

---

## CorrectnessCheck

**Definition:** Automated verification that generated code is syntactically valid and meets task-specific acceptance criteria (parser, lint, optional execution).
**Code name:** `CorrectnessCheck`

---

## DistillationPipeline

**Definition:** End-to-end flow: gold seeds → synthetic expansion → quality filter → train student → benchmark → report.
**Code name:** `DistillationPipeline`

---

## PipelineOperator

**Definition:** Human who runs the local pipeline, reviews samples, and approves dataset publication and training runs.
**Not the same as:** End-user of generated code (out of scope for phase 1).
**Code name:** `PipelineOperator`

---

## DatasetVersion

**Definition:** Immutable identifier for a published `SyntheticDataset` snapshot (e.g. `ds-2026-06-20-v1`).
**Code name:** `DatasetVersion`

---

## ExperimentPhase

**Definition:** One of the five major project phases documented in `PHASES.md`.
**Enum values:** `SyntheticDataGeneration`, `StudentModelSelection`, `TrainingComparison`, `Evaluation`, `Reporting`
**Code name:** `ExperimentPhase`
